## What is UV?

**UV** is a modern, blazingly fast Python package and project manager written in Rust. It's designed to replace multiple tools with a single unified solution:

| Traditional Tools | UV Replacement |
|------------------|----------------|
| pip | ✅ `uv add` |
| pip-tools | ✅ `uv lock` |
| virtualenv/venv | ✅ Automatic `.venv` |
| poetry/pipenv | ✅ Built-in project management |
| pyenv | ✅ `uv python` |

### Key Benefits
- ⚡ **10-100x faster** than pip (written in Rust)
- 🔒 **Secure** with SHA256 hash verification
- 📦 **All-in-one** tool for Python management
- 🔄 **Reproducible** environments across machines
- 🎯 **Modern** standard following PEP 621

## Understanding the Files

### `pyproject.toml` - Your Project's Blueprint

**Purpose**: Declares what packages your project needs (high-level requirements)

```toml
[project]
name = "my-project"
version = "1.0.0"
description = "My Python project"
requires-python = "==3.13.*"

dependencies = [
    "requests",      # Latest compatible version
    "pyats",         # No version constraint
    "fastapi",       # Allows automatic updates
]

[dependency-groups]
dev = [
    "pytest",        # Development/testing tools
    "ruff",          # Linters
    "ipython",       # Debuggers
]
```

**Characteristics:**
- ✅ **Human-readable** - You edit this directly
- ✅ **High-level** - Lists only direct dependencies
- ✅ **Flexible** - Loose or no version constraints
- ✅ **Intent-focused** - What your code actually uses
- ✅ **Maintainable** - Easy to update and manage

**Think of it as:** Shopping list ("I need milk, eggs, bread")

---

### `uv.lock` - Exact Dependency Snapshot

**Purpose**: Records exact versions of every package (including all sub-dependencies)

```toml
[[package]]
name = "requests"
version = "2.32.5"
source = { registry = "https://pypi.org/simple" }
dependencies = [
    { name = "certifi" },
    { name = "charset-normalizer" },
    { name = "idna" },
    { name = "urllib3" },
]
sdist = { url = "...", hash = "sha256:..." }
wheels = [
    { url = "...", hash = "sha256:..." },
]

[[package]]
name = "certifi"
version = "2025.10.5"
...
```

**Characteristics:**
- 🔒 **Auto-generated** - Never edit manually
- 🔒 **Complete** - Every package + sub-dependencies
- 🔒 **Exact** - Pinned versions with hashes
- 🔒 **Secure** - SHA256 verification
- 🔒 **Reproducible** - Same environment everywhere

**Think of it as:** Receipt ("Brand X milk 1L barcode#123, purchased at 2:34pm")

---

### `requirements.txt` - Pip-Compatible Export

**Purpose**: Export format for pip compatibility (auto-generated from `uv.lock`)

```txt
# This file was autogenerated by uv
requests==2.32.5
certifi==2025.10.5
charset-normalizer==3.4.4
idna==3.11
urllib3==2.5.0
pyats==25.9
pyats-aereport==25.9
pyats-aetest==25.9
...
```

**Characteristics:**
- 📝 **All packages** - Direct + indirect (150+ packages)
- 📝 **Pinned versions** - Exact version numbers
- 📝 **Flat list** - No dependency tree structure
- 📝 **Legacy compatible** - Works with pip

**Generated with:**
```bash
uv export --no-hashes --format requirements-txt > requirements.txt
```

---

### File Comparison Table

| Feature | `pyproject.toml` | `uv.lock` | `requirements.txt` |
|---------|-----------------|-----------|-------------------|
| **Edit manually?** | ✅ Yes | ❌ No | ❌ No |
| **Number of packages** | ~25 (direct) | ~150 (all) | ~150 (all) |
| **Version format** | Loose/none | Exact+hash | Exact |
| **Purpose** | Project config | Lockfile | Pip export |
| **Update method** | Manual edit | `uv lock` | `uv export` |

## Creating a New Project

### Method 1: Start from Scratch

```bash
# Navigate to parent directory
cd ~/DevnetExpert

# Create new project
uv init mock3

# Enter project directory
cd mock3

# Pin Python version
uv python pin 3.13.9

# Add dependencies
uv add requests pyats genie netmiko

# Add dev dependencies
uv add --dev pytest ruff ipython

# Verify installation
uv pip list
```

---

### Method 2: Clone Existing Project Configuration (RECOMMENDED)

```bash
# Navigate to parent directory
cd ~/DevnetExpert

# Create new project structure
uv init mock3
cd mock3

# Copy existing configuration files
cp ../pyproject.toml .
cp ../uv.lock .

# Pin Python version
uv python pin 3.13.9

# Install everything with exact versions
uv sync

# Verify
uv run python -c "import pyats; import genie; print('Success!')"
```

**This is the BEST method for CCIE Automation because:**
- ✅ Uses proven working configuration
- ✅ Installs exact versions from exam requirements
- ✅ Guarantees identical environment
- ✅ Fastest setup (just copy + sync)

---

### Method 3: From requirements.txt

```bash
cd ~/DevnetExpert
uv init mock3
cd mock3

# Pin Python version
uv python pin 3.13.9

# Install from requirements.txt
uv pip install -r ../requirements.txt

# Generate lockfile
uv lock
```

## Installing Dependencies

### Understanding `uv sync`

**The most important command for installing dependencies:**

```bash
uv sync
```

**What it does:**
1. ✅ Reads `uv.lock` file
2. ✅ Installs ALL packages with exact versions
3. ✅ Installs ALL dependencies (direct + indirect)
4. ✅ Creates `.venv` virtual environment
5. ✅ Verifies package integrity using SHA256 hashes
6. ✅ Installs 150+ packages in seconds

**When to use:**
- Setting up a new development environment
- After cloning a repository
- After someone updates `uv.lock`
- To restore exact working state

---

### Adding New Packages

```bash
# Add a single package
uv add requests

# Add package with version constraint
uv add "requests>=2.32.0"

# Add package with exact version
uv add "requests==2.32.5"

# Add package with extras
uv add "scrapli[asyncssh,ssh2,ttp]"

# Add multiple packages
uv add requests httpx fastapi

# Add development dependencies
uv add --dev pytest ruff mypy

# Add and upgrade existing package
uv add --upgrade requests
```

**What happens:**
- Updates `pyproject.toml`
- Updates `uv.lock` with resolved dependencies
- Installs the package in `.venv`

---

### Removing Packages

```bash
# Remove a package
uv remove requests

# Remove dev dependency
uv remove --dev pytest
```

## Essential UV Commands

### Project Management

```bash
# Initialize new project
uv init project-name

# Initialize in current directory
uv init

# Sync environment with lockfile
uv sync

# Sync and update all packages
uv sync --upgrade

# Lock dependencies without installing
uv lock
```

---

### Python Version Management

```bash
# Install Python version
uv python install 3.13.9

# List installed Python versions
uv python list

# Pin Python version for project
uv python pin 3.13.9

# Check pinned version
cat .python-version
```

---

### Package Management

```bash
# Add packages
uv add package-name
uv add "package-name==1.2.3"
uv add --dev package-name

# Remove packages
uv remove package-name

# Update specific package
uv add --upgrade package-name

# Update all packages
uv sync --upgrade
```

---

### Running Commands

```bash
# Run Python script (without activating venv)
uv run python script.py

# Run any command in venv
uv run pytest
uv run python -m flask run

# Run Python code directly
uv run python -c "import sys; print(sys.version)"

# Activate venv manually (traditional way)
source .venv/bin/activate
```

---

### Inspecting Environment

```bash
# List installed packages
uv pip list

# Show package details
uv pip show package-name

# List outdated packages
uv pip list --outdated

# Show dependency tree
uv tree

# Export to requirements.txt
uv export --format requirements-txt > requirements.txt

# Export without hashes
uv export --no-hashes --format requirements-txt > requirements.txt
```

## Best Practices

### For CCIE Automation / Certification Prep

1. **Use exact versions** - Match exam specifications precisely
   ```toml
   requires-python = "==3.13.*"
   ```

2. **Preserve working configurations** - Don't update unless necessary
   ```bash
   # Don't run this unless you have a reason:
   # uv sync --upgrade
   ```

3. **Copy proven setups** - Use existing `uv.lock` for new projects
   ```bash
   cp ../uv.lock ./mock3/
   uv sync
   ```

4. **Keep separate environments** - Each mock exam gets its own project
   ```bash
   DevnetExpert/
   ├── mock1/
   ├── mock2/
   └── mock3/
   ```

---

### General Development

1. **Commit lockfiles** - Always commit `uv.lock` to version control
   ```bash
   git add pyproject.toml uv.lock
   ```

2. **Use loose versions in pyproject.toml** - Let UV handle specifics
   ```toml
   # Good
   dependencies = ["requests"]
   
   # Only if needed for compatibility
   dependencies = ["requests>=2.30.0"]
   ```

3. **Run `uv sync` regularly** - Keep environment synchronized
   ```bash
   # After pulling changes
   git pull
   uv sync
   ```

4. **Use `uv run` for scripts** - Avoid manual venv activation
   ```bash
   # Instead of:
   source .venv/bin/activate
   python script.py
   
   # Do:
   uv run python script.py
   ```

## Workflow Comparison

### Traditional vs UV

| Task | Traditional | UV |
|------|------------|-----|
| **Create venv** | `python -m venv .venv` | Automatic |
| **Activate venv** | `source .venv/bin/activate` | `uv run` (or auto) |
| **Install package** | `pip install requests` | `uv add requests` |
| **Pin versions** | `pip freeze > requirements.txt` | `uv.lock` (auto) |
| **Install from file** | `pip install -r requirements.txt` | `uv sync` |
| **Update package** | `pip install --upgrade requests` | `uv add --upgrade requests` |
| **List packages** | `pip list` | `uv pip list` |

## Troubleshooting

### Common Issues

**Issue: "No Python interpreter found"**
```bash
# Install Python via UV
uv python install 3.13.9

# Pin it to project
uv python pin 3.13.9
```

---

**Issue: "Package conflicts"**
```bash
# Remove lockfile and regenerate
rm uv.lock
uv lock

# Or force sync
uv sync --force
```

---

**Issue: "Virtual environment not activated in VS Code"**
```bash
# Manually select interpreter
# Ctrl+Shift+P -> "Python: Select Interpreter"
# Choose: .venv/bin/python

# Or restart VS Code
```

---

**Issue: "Packages installed but imports fail"**
```bash
# Verify you're using the right Python
uv run python -c "import sys; print(sys.executable)"

# Should show: /path/to/project/.venv/bin/python

# Reinstall if needed
rm -rf .venv
uv sync
```
