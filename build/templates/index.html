RAG behavior in v1.8
        
When "Use RAG" is checked in agentic chat, the server fetches top-5 corpus chunks for your prompt and prepends them to the system prompt before the agent loop starts. The agent's search_corpus tool is still available for follow-up queries.

      

      
Stop button behavior
        
Clicking Stop during an agent run posts to /api/agent-cancel with the run_id. The agent loop checks the cancellation flag at every iteration boundary and tool-dispatch boundary. Any tool call already in-flight will finish; the loop exits cleanly with stopped_reason="cancelled" immediately after.

      

      
RESTCONF YANG path on IOS-XE — common gotchas
        

          OSPF: use Cisco-IOS-XE-native:native/router/router-ospf on IOS-XE 16.6+ (NOT router/ospf). The standalone Cisco-IOS-XE-ospf module may not be installed on all images.

          BGP: Cisco-IOS-XE-native:native/router/router-bgp

          To verify what's available on YOUR device:
          curl -k -u USER:PASS https://<host>/restconf/data/ietf-yang-library:modules-state

          How to add devices
          

            Inventory file: —

            Edit it directly, then click ↻ Reload above (no service restart needed). Two formats are supported:


            Flat (simplest):

            
devices:
  - name: cat8Kv71
    host: 192.168.89.71
    port_netconf: 830
    port_ssh: 22
    username: expert
    password: $env:CAT8KV71_PASSWORD
    device_type: cisco-iosxe
    read_only: false
    default: true

            pyATS testbed:

            
devices:
  cat8Kv71:
    os: iosxe
    credentials:
      default:
        username: "%ENV{CAT8KV71_USERNAME}"
        password: "%ENV{CAT8KV71_PASSWORD}"
    connections:
      cli:     {protocol: ssh,     ip: "%ENV{CAT8KV71}", port: 22}
      netconf: {protocol: netconf, ip: "%ENV{CAT8KV71}", port: 830}
    custom:
      read_only: false
      default: true

            For env-var passwords, set them in the systemd drop-in at ~/.config/systemd/user/hitech_automation_ai.service.d/secrets.conf and run systemctl --user restart hitech_automation_ai.
